Vue2项目使用Vite替代Vue-Cli

2022-03-29 Comments 前端工程化

背景

目前在公司使用的是 Thinkpad T450 的老电脑，随着页面和依赖的增多，前端项目的启动速度不断变慢，因此用 Vite 替代原先的 Vue-Cli。Vue-Cli 使用 Webpack 作为构建工具，与 Vite 显著的区别在于：Webpack 在开发环境启动时会解析依赖树并对所有的依赖进行处理，而 Vite 省略了开发环境的打包过程，利用浏览器解析 import 语句中的模块向 Vite 本地服务发起请求，再按需编译返回，因此不会随着页面数量的增大而增加编译时间。

原理

网上已经有很好的总结，esbuild 以及上文提到的按需编译是大杀器，详见Vite 介绍及实现原理

方案

目前 vite 在生产环境打包采用 Rollup，产物与开发环境表现不能保证完全一致，考虑稳定性，且编译产物已放到 CI/CD 中，故只在本地开发时使用 Vite。

实现

修改其实不多。新建vite.config.ts并在项目中引入 Vite 相关依赖，做了如下修改：

  1. 使用 vite-plugin-externals 实现 webpack 的 externals 配置
  2. 使用 vite-plugin-html 实现 html 支持 ejs 模板并注入变量
  3. 引入 vite-plugin-vue2 支持 Vue2
  4. 修复@origin/vite-plugin-require-context 插件处理不了逗号的问题，这个插件的原理就是解析文件匹配正则表达式，然后读取文件列表，最后转化为 import 语句并插入到代码中。
  5. import vue 文件加上.vue 后缀
  6. 将 node_modules 中 commonjs 格式的模块加入 optimizeDeps.include
  7. 将 src 中的 module.exports 以及 require 表达式改写为 import 和 export

效果

第一次启动速度 4365ms

第二次 1932ms
