程序员的成长

2016-03-29 Comments Life

看了前端的自我成长，想起Winter老师在第二届FEDay上讲了这么一句话——“好前端才分对错，成年人只分利弊”，这个一下子引起我想到很多。自己亲身经历了一些前端开发，前端的确并不难学，没有硬性的算法要求，拥有众多的实现方法和较大容错性，这使得很多前端都满足于界面能看能用，而不去考虑如何优化，对于一些细节也靠无思考地尝试去解决，而不深究原理。事实上每一门语言都有其语法及规范，所以理论上都是有对和不对的代码写法，虽然实际上并不一定有，但是应该去探寻，追根溯源的过程收获比单纯会用多很多；作为前端遇到如js闭包跨域的问题只拿现成解决方案来用而不去想其中的实现原理、规则意义和底层特性，那么以后遇到变化的问题就要重新浪费一遍时间去尝试寻找，毫无提升。所以就引出了这篇文章的主题，如何去“深度”学习，了解事物的本质一面。

现在很多公司都说缺前端，然而另一个情况是有很多人都投前端岗位，也有人说前端现成的框架那么多，覆盖面也很广，还需要招那么多人来开发吗？其实公司更多是需要独立或自定义地开发前端，不能只依赖现成框架。然而大多数人只是会用框架，对于底层了解不多，无法胜任更高的工作要求，自身也没有进一步学习，重复已经掌握的技能。这些人就是前面说的满足于能看能用而不了解原理的“码农”，只是单纯的代码搬运工，对于公司的价值不大，甚至还可能带来负价值。商业公司以价值最大化作为目标，宁可加班也不会招这些人。

所以一个前端开发者应该有思想，能够思考遇到的问题总结出一些观点，想办法去改进或去查阅资料解决，可以的话，甚至可以自己提供一个解决方案。JavaScript为什么这几年这么火？主要的原因在于Node.js，原先js在浏览器端就已经拥有数量庞大的开发者，Node赋予了前端开发者用js写后台的能力，使得大多数很难接触后台的前端工程师也能搭建完整web应用，并且催化了前后端分离和很多自动化工具，大大提高了开发效率，然而Node的应用场景还有局限，稳定性和安全性看似通过多进程和集群已经解决了，但为什么真正的后台还是没有用，这里就不只是技术的原因了，成本也是要考虑的；其次则因为js是一门很容易上手的弱类型语言，降低了门槛，而且在浏览器端无人能敌，这也是开发者众多的原因。

另一个要求则是要细致，前端不难，但是细节很多，对于细节的注意能发现许多本质性的东西，这可以说是幸运也是不幸，因为锻炼了观察能力和细心的同时带来许多繁琐。作为一个有一定经验的程序员来说，我经常会去关注用户体验的问题，对于网页的一些交互细节、配色和布局，我虽然不是很专业，但也会自己去试用改进；而对于网页性能优化，在看似合理的原则下，其实也有很多细节要处理，比如为减少加载时间通常网页都要合并静态资源减少请求，然而这对于开发者来说其实并不方便，要增加一个额外合并的工作，如果考虑自动化部署，开发完自动合并部署的话，对于同域不同页面，如果js文件有一些不同，会导致重新合并和全新请求，这会造成服务器的更大负担，所以，这些问题其实考虑包括用户习惯和运行环境在内的多种因素，已经不只是前端范畴了。

一晃三年

2015-12-15 Comments Life

一晃三年，历历眼前。时光又仿佛回到三年前，12月30日，体育馆二楼，《仰望星空》的歌声响起，高中最后一个艺术节晚会落幕。当时的我在想什么？迷茫？涅槃？光荣？梦想？绝望？恐惧？这些都不重要，一切都如流水一般卷过，不留下一点余地……

于是，来年六月，在生活被强行切开插入插曲之后，走上高考的考场，此刻的心情是怎样？如死一般的平静，穿云拨雾的坚定。于是，来到了华农。

三年来，许多都变了，无奈兼顾各方各面的因素，遇到许多有趣的人和事，也见识过无数魑魅魍魉。最后还是发现，竟回到了原点，平行的线无法弯曲，相交的线只有一个交点，一切努力付诸流水，一切心念化为花火。

前段时间在人脉看到辜新宇的文章《Time Will Tell》，开篇就是：

「焦虑」是个流行词。 从华附毕业的人，大都有一个势如破竹的少年，和一个焦虑摇摆的青年——也许是因为害怕没办法再成为一个「好学生」吧。如果挂科了，会找到好工作吗？不在哈佛耶鲁，也没去香港，留在中国是正确的吗？中国的大学贬值越来越厉害，而如果随波逐流，会不会再也没有机会绝地反击了？率性而为、听从内心召唤是最优选吗？

幸运，在一切都还来得及的时候。一直想往“正路”走，从小到大，然而骨子里总是有按捺不住做“坏学生”的本能，不愿成为湮没在历史中的一粒尘埃。本来就不是一路人，何必强融。但又何妨，Time Will Tell，吾道不孤。我还有我的母校，我的挚友，我的家人。

一直相信事在人为，万事不可轻言放弃。现在看来则应加上，我命由我不由天，于无声处听惊雷。无可惜，不要让自己后悔便好。

「我越来越觉得『事在人为』这四个字是有道理的。大部分时候，我们之所以做不到还是因为自己不够勇敢，不够努力。尤其当我看到身边同样家境一般的朋友，他们却有着坚定的出国信念，他们一直在搜集信息，同时不断努力，最终拿到了奖学金与经费。他们就是能出去。我一直都很想出去看看，但是我就是早早地告诉自己，我就是不能，所以我真的不能，我连尝试都不敢。」

现在看来真是感慨万千。

总有人会四处与你为难，总有人会投来敌视的目光，但这些都不重要，你要做的只是保持一颗冷静坚定的心，执着地寻找一直想要的东西，然后踏平一切的阻碍，也许会得罪很多人，但又有几个人对你真正重要呢？也许都无法在生命结束前做到，但万一，做到了呢？

吾道不孤

「『见到一座山,就想知道山后面是什么，但即使还是山，他还是要亲自爬过去才甘心。』哪怕过去的你多么糟糕，也不妨碍你成为一个很棒的人，请坚持下去，时间会告诉你答案。至少你要相信，你可是从华附出来的。」

JavaScript类型

2015-02-26 Comments JavaScript

Javascript类型是一个很奇妙的东西，分为原始类型（primitive）和对象类型（Object）两大类。

如果使用typeof操作符对变量进行检查，会发现有这些结果：boolean, number, string, undefined, object, function。但这些不能准确反映Javascript的类型，其中function看似是一种独立的“函数”类型，严格意义上却是一种特殊的对象，不是一种数据类型，但其拥有的一些特殊属性也使我们有必要把它区分开来。

根据最新ECMAScript标准，其实真正的类型分为如下几种:

  • Boolean
  • Null
  • Undefined
  • Number
  • String
  • Symbol (new in ECMAScript 6)
  • Object

上面前6种为原始类型，原始类型的一个特点是其值一旦创建便不可改变。虽然对于常见编码影响不大，但这一点是很少被注意到的。

在原始类型中，Undefined类型只有undefined一个特殊值。在变量声明却未初始化时其值就是undefined，它是全局对象的一个属性。

Null是另一个只有一个字面值null的数据类型。null指向一个空的对象，所以对其typeof会显示object，然而它不是全局对象的一个属性。null与undefined关系紧密，然而他们的用法不同，undefined是作为一个默认的空值存在，而null更多是作为对象占位符，表示需要传递一个空值。
