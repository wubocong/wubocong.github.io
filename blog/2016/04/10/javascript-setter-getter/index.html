JavaScript setter&getter

2016-04-10 Comments JavaScript

这篇文章主要探讨object的setter和getter。之所以用object而不是Object，是因为Object是JavaScript中一个构造函数对象,而不是一种类型，不可混淆。

setter和getter是对象属性的一种特殊用法，getter方法不接受任何参数，返回一个值；setter方法只接受一个参数，不返回任何值。在ES5中，有两种定义的方法：

可以看到，getter和setter通常成对出现（但不是必须），且一般不与其他四种特性一同设置（会覆盖对属性原有value值的操作）。并且即使一个object处于frozen状getter和setter都可以自由设置，不受影响。

所以，从本质上理解，getter就是用来获取object内部的一些值，setter就是用来设置object内部的一些值，这仿照了其他高级语言（C++, Java）中的公有方法，可以模实现分封装的特性，对于规范工程化开发有不少帮助。但是因为JavaScript的object并不存在私有成员这一特征，所以始终可以直接从外部访问内部成员，是伪封装。要实真正的封装，目前只有通过闭包。
