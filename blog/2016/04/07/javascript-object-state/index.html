JavaScript中object的三个状态

2016-04-07 Comments JavaScript

object类型的对象有三种状态（注意是状态而不是属性）：extensible, sealed, frozen。默认情况下，object对象是可扩展的：即可以为他们添加新的属性，且更改它们的 __proto__ 属性。具体如下

  1. extensible是指它可扩展。
  2. sealed是指它不可扩展，且所有自身属性都不可配置的（non-configurable）。
  3. frozen是指它不可扩展，所有属性都是不可配置的（non-configurable），且所有数据属性（data properties）都是不可写的（non-writable）。数据属性是指那些没有取值器（getter）或赋值器（setter）的属性。

这里有一个概念——可配置。它是指对象属性的这六个额外设置选项（我没打错）：configurable, enumerable, value, writable, get, set可以用Object.defineProperty()进行修改。configurable即可配置，如果将其设置为false则这六个值以后都无法再修改；enumerable指的是该属性是否可以被枚举出来（for…in循环、Object.keys和Object.getOwnPropertyNames()），要注意即使为false通过“.”运算符仍然能访问，这也是符合常理的，类似于封装；value是指其值；writable指其是否可被修改。get，set指该属性取值器和赋值器的方法内容，请看我的这篇文章JavaScript setter&getter。

对应的，Object构造函数对象里有三种方法来使他们转变成各自状态（ Object.preventExtensions(), Object.seal(), Object.freeze() ）和三种方法（ Object.isExtensible(), Object.isSealed(), Object.isFrozen() ）来判断一个对象是否处于某种状态。

在此基础上，我们进行三个实验：

  1. 对object对象使用 Object.preventExtensions()，其变为extensible状态，即无法添加新属性，但可以修改、删除和配置已有属性。
  2. 对object对象使用Object.seal()，则其变为sealed状态，无法添加新属性也无法删除和配置已有属性，只能修改已有属性。
  3. 对object对象使用Object.freeze()，则其变为frozen状态，无法添加、删除、修改和配置任何属性。

可见Object.preventExtensions()，Object.seal() 和 Object.freeze() 方法都可以标记一个对象为不可扩展，他们的封闭程度逐级递增，所以frozen必然是sealed的，sealed也必然是inextensible的。
